# Prompts

A prompt is simply the input you give to an LLM to get a response. It might be a question or a command, or just a casual comment to start a conversation. The model responds to a prompt with a completion.

## Types of Prompt

There are two main types of prompts:

- **System prompts** that set the behavior and tone of the model, and any constraints it should adhere to. For example, "You're a helpful assistant that responds in a cheerful, friendly manner.". System prompts determine constraints and styles for the model's responses.
- **User prompts** that elicit a response to a specific question or instruction. For example, "Summarize the key considerations for adopting generative AI described in GenAI_Considerations.docx for a corporate executive. Format the summary as no more than six bullet points with a professional tone.".

Usually, the system prompt is set by the application that uses the model. User prompts can be entered by a human user in a chat application; or in some cases generated by the application on the user's behalf. The model responds to user prompts while obeying the overall guidance in the system prompt.

## Conversation History

To keep a conversation consistent and relevant, generative AI apps often keep track of the conversation history; and include summarized versions of it in subsequent prompts. This ensures there's an ongoing context for the conversation that the model can build on.

For example, suppose the model responds to the system and user prompts described previously with the following completion:

&gt; Key considerations for adopting Generative AI include:
&gt;
&gt; - Establish Clear Governance and Responsible AI Policies
&gt; - Prioritize High-Value Use Cases Across Corporate Functions
&gt; - Mitigate Risks Around Privacy, Security, and Compliance
&gt; - Foster Organizational Readiness and Cultural Adaptation
&gt; - Measure Impact and Build Accountability
&gt; - Stay Ahead of Strategic and Competitive Shifts

You could respond with a follow-up question, such as "What are common privacy-related risks?". The prompt would include the new question, but also the previous prompts and responses; providing context to the model so that it understands the question in relation to Generative AI adoption.

## Retrieval Augmented Generation (RAG)

To add even more context, generative AI applications can use a technique called retrieval augmented generation (RAG). This approach involves retrieving information, like documents or emails, and using it to augment the prompt with relevant data. The response generated by the model is then grounded in the information that was provided.

For example, suppose you submit a prompt like "What's the maximum I can claim for travel expenses on a business trip?". With no other information, a model will respond with a generic answer - probably telling you to consult your organization's expenses policy documentation. A better solution would be to build an expenses assistant app that initially queries the organization's expenses policy documentation, retrieving sections related to "travel expenses"; and then includes the retrieved information in the prompt that is sent to the model, along with your original question. Now the model can use the expenses policy information in the prompt to provide context, and respond with a more relevant answer.

## Tips for Better Prompts

The quality of responses from generative AI assistants not only depends on the language model used, but on the prompts you submit to it.

To get better results from your prompts:

- **Be clear and specific** – prompts with explicit instructions or questions work better than vague language.
- **Add context** - mention the topic, audience, or format you want.
- **Use examples**, If you want a certain style, provide an example of what you mean.
- **Ask for structure**, Like bullet points, tables, or numbered lists.

Using well-designed prompts can make a huge difference to the results you'll get from your generative AI model.

## Use Cases and Solutions

### System and User Prompt Use Cases

- **Maintaining consistent brand voice** - A company wants all AI-generated customer service responses to sound professional and empathetic
  - *Solution:* Configure a system prompt that sets the tone ("You are a professional, empathetic customer service assistant") while allowing user prompts to handle specific customer inquiries within those constraints

- **Separating concerns in multi-user applications** - An application serves multiple users with different needs but requires uniform behavior standards
  - *Solution:* Use system prompts to enforce global constraints (safety guidelines, response format) while allowing each user's specific questions or commands to be handled through dynamic user prompts

### Conversation History Use Cases

- **Handling follow-up questions** - A user asks "What are common privacy-related risks?" after receiving a summary about AI adoption considerations
  - *Solution:* Include summarized previous prompts and completions in subsequent prompts so the model understands "privacy-related risks" refers specifically to AI adoption, not general cybersecurity

- **Maintaining context across long sessions** - A user engages in a 20-turn conversation about project planning
  - *Solution:* The application tracks conversation history and includes relevant previous exchanges in each new prompt, ensuring the model remembers earlier decisions and constraints mentioned by the user

### Retrieval Augmented Generation (RAG) Use Cases

- **Organization-specific policy questions** - An employee asks "What's the maximum I can claim for travel expenses on a business trip?"
  - *Solution:* Build an assistant that queries the organization's expenses policy documentation, retrieves sections related to "travel expenses", and includes that specific information in the prompt alongside the user's question, grounding the response in actual policy rather than generic advice

- **Email-based customer support** - A support agent needs to respond to a complex inquiry referencing previous communications
  - *Solution:* Use RAG to retrieve relevant emails or documents related to the customer's issue, augment the prompt with this retrieved context, and generate a response informed by the specific history and details of the case

### Prompt Engineering Use Cases

- **Getting structured output** - A manager needs executive summary data in a scannable format
  - *Solution:* Add explicit structure requests to the prompt, such as "Format the summary as no more than six bullet points with a professional tone" rather than asking for a generic summary

- **Achieving specific writing styles** - A content creator wants marketing copy that matches their brand's casual, witty tone
  - *Solution:* Provide an example of the desired style in the prompt ("Here's an example of our tone: [sample text]. Write a product description in this style.") rather than using vague adjectives like "fun" or "engaging"

- **Reducing ambiguous responses** - A researcher gets vague answers when asking about technical topics
  - *Solution:* Be clear and specific in the prompt by adding context about the topic, audience, or format needed, rather than using open-ended questions that leave interpretation to the model
